# Experiment Portal — Предложения по улучшению

## Высокий приоритет

### 1. Увеличение времени жизни сессии (JWT token)
Сейчас при долгом отсутствии активности на странице происходит частый разлогин.
Необходимо увеличить TTL JWT-токенов и/или реализовать механизм refresh-токенов
для продления сессии без полного перелогинивания.

### 2. Генерация TypeScript SDK из OpenAPI
Типы в `src/types/index.ts` (481 строка) поддерживаются вручную, хотя есть `make generate-sdk`.
Рассинхронизация с бэкендом — вопрос времени. Подключить автогенерацию в CI.

### 3. Клиентская валидация форм (zod)
Сейчас валидация почти полностью на бэкенде. Добавить `zod` для форм создания
эксперимента, сенсора, вебхуков — даст и валидацию, и типы из одной схемы.

### 4. Real-time обновления статусов
Статусы Run / CaptureSession обновляются только при ручном перезаходе или через React Query
refetch. Добавить WebSocket / SSE push для статусов, чтобы UI не показывал stale-данные.

### 5. Виртуальный скроллинг для длинных списков
Списки экспериментов/ранов при 1000+ элементов будут тормозить.
`@tanstack/react-virtual` решает это минимальными усилиями.

## Средний приоритет

### 6. Dark mode
Дизайн-токены уже вынесены в CSS-переменные — осталось добавить второй набор через
`prefers-color-scheme` или ручной переключатель в хедере.

### 7. Shared component library (`common/`)
`common/` и `libs/` пустые. Компоненты `StatusBadge`, `Modal`, `Pagination`, `Loading`,
`EmptyState` используются повсюду — вынести в `common/` как переиспользуемый пакет.

### 8. Консистентность loading-состояний
Где-то скелетоны, где-то спиннеры — унифицировать подход (рекомендация: skeleton screens
для списков, spinner для одиночных операций).

### 9. Storybook для компонентов
~15 переиспользуемых компонентов — документация через Storybook ускорит разработку
и ревью, особенно при онбординге новых участников.

## Низкий приоритет

### 10. Accessibility (WCAG 2.1 AA)
ARIA-атрибуты есть частично, но модалки не все обрабатывают focus trap и Escape.
Добавить `focus-trap-react` или аналог для всех модальных окон.

### 11. Error tracking (Sentry)
В продакшене ошибки уйдут в никуда — подключить Sentry или аналог
для отлова runtime-ошибок.

### 12. PWA / Offline support
Service worker для кеширования статики и очереди запросов.

### 13. Расширение E2E-покрытия (Cypress)
Cypress настроен, но тестов мало. Покрыть основные сценарии:
логин → создание эксперимента → запуск рана → просмотр телеметрии.

### 14. Code splitting
Бандл JS > 5 МБ. Добавить `React.lazy()` + `Suspense` для страниц,
чтобы уменьшить начальную загрузку (особенно `plotly.js-dist-min`).
